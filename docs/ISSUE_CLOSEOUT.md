# GitHub Issue Closeout — Mini Guide

## Auto-Close (ideal)
- In PR Description: `Closes #10` / `Fixes #16`
- Wichtig: funktioniert zuverlässig, wenn PR in den **Default Branch** gemerged wird.

## Manuell schließen (ok, wenn schon gemerged)
- Wenn du bereits nach `main` gemerged hast und die PR keine Closing Keywords hatte:
  - Issues öffnen → kurz checken → Close.
  - Kommentar rein: „Done in main (commit/PR …)“.

## Warum das sinnvoll ist
- Dein Board bleibt sauber, und du siehst sofort, was wirklich noch fehlt.
